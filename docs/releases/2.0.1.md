# Release 2.0.1

## Summary
This release establishes the public Chromium Manifest V3 continuation of TIFF Viewer.

It keeps the original project lineage intact, preserves attribution, and introduces a maintained Chromium package that works on current MV3-based browsers.

Published Chrome Web Store listing:
- `https://chromewebstore.google.com/detail/tiff-viewer/dldbfebnfnefnklickhbgefihbldljok`

## Highlights
- Added a dedicated `chrome-mv3/` package for Chromium browsers
- Replaced the old MV2 request interception model with an MV3-compatible service worker and offscreen rendering flow
- Added a dedicated viewer for direct `.tif` and `.tiff` navigations
- Added inline TIFF replacement for unsupported images on web pages
- Preserved the upstream decoder and options model where practical
- Added attribution, migration notes, and maintainer-facing project documentation

## Technical changes
- `background.js` in the Chromium package now coordinates redirects, response detection, and render requests
- `offscreen.js` performs TIFF decoding in an offscreen document rather than a persistent background page
- `content/image-handler.js` replaces broken or unsupported TIFF images inline
- `viewer/` provides the direct-navigation rendering experience
- `options/options.js` was updated to respect MV3 runtime boundaries

## Documentation included
- Root README rewritten as a fork front page
- `MIGRATION_NOTES.md` documents the MV2 to MV3 migration
- `docs/CHROME_WEB_STORE_LISTING.md` provides store-ready listing copy

## Notes
- The original repository root remains in place as historical upstream source
- The maintained Chromium package lives in `chrome-mv3/`
- Decoder limitations inherited from the bundled TIFF library still apply
