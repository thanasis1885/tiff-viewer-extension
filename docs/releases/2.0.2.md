# Release 2.0.2

## Summary
This update focuses on the Chromium viewer experience for document-style TIFF files.

It keeps the Manifest V3 rendering pipeline from `2.0.1`, while making the direct TIFF viewer cleaner, easier to navigate, and better suited to tall scanned documents.

Chrome Web Store listing:
- `https://chromewebstore.google.com/detail/tiff-viewer/dldbfebnfnefnklickhbgefihbldljok`

## Highlights
- Removed the large viewer header so documents open closer to the top of the page
- Added a compact sticky toolbar with Back, zoom, Fit Width, 100%, Print, and Open Original controls
- Added a Print action that prints the rendered TIFF without the viewer UI
- Improved button typography and toolbar styling for clearer rendering
- Changed the document layout to use a single vertical page scrollbar
- Preserved horizontal scrolling only when zooming beyond the available page width

## Technical changes
- Updated `chrome-mv3/viewer/viewer.css` so the TIFF viewport grows with the document instead of using an internal fixed-height scroll area
- Updated `chrome-mv3/viewer/viewer.js` so Fit Width measures the actual content width after the layout change
- Kept the existing MV3 service worker, offscreen renderer, permissions, and decoder behavior unchanged

## Notes
- This is a user-interface update only
- No new permissions are introduced
- Existing TIFF decoder limitations still apply
